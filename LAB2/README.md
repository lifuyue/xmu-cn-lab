# LAB2

RS232C 7-bit 电压序列编码与解码实验。

## 实验目标

实现题目要求的两个接口：

```cpp
int rs232c_encode(double *volts, int volts_size, const char *msg, int size);
int rs232c_decode(char *msg, int size, const double *volts, int volts_size);
```

并通过命令行程序和自动化测试验证编码与解码过程是否正确。

## 目录结构

```text
include/rs232c.hpp      公共接口定义
src/rs232c.cpp          核心编码与解码实现
src/rs232c_cli.cpp      命令行演示程序
tests/test_rs232c.cpp   自动化测试
Makefile                构建脚本
```

## 实现约定

- 使用标准 RS232C 极性：
  - 逻辑 `1`、空闲位、终止位 -> `-12.0V`
  - 逻辑 `0`、起始位 -> `+12.0V`
- 每个字符固定编码为 10 个电压值：
  - `空闲位 + 起始位 + 7 位数据 + 终止位`
- 位序为位小端序：
  - 数据位按 `LSB -> MSB` 输出
- 字节顺序为消息原始顺序：
  - 多字符字符串按从左到右的顺序依次编码
- 只接受 7-bit ASCII 字符

## 编码示例

把字母 `A` 编码成电压序列：

```bash
./bin/rs232c_demo encode A
```

预期输出：

```text
-12 12 -12 12 12 12 12 12 -12 -12
```

说明：

- `A` 的 7-bit ASCII 是 `1000001`
- 位小端序输出后，数据位顺序是 `1 0 0 0 0 0 1`
- 因此完整帧为：
  - 空闲位 `1`
  - 起始位 `0`
  - 数据位 `1 0 0 0 0 0 1`
  - 终止位 `1`

## 解码示例

把一组电压解码回字符：

```bash
./bin/rs232c_demo decode -12 12 -12 12 12 12 12 12 -12 -12
```

预期输出：

```text
A
```

## 返回值约定

- `rs232c_encode`
  - 成功时返回实际写入的电压数量
  - 失败时返回 `-1`
- `rs232c_decode`
  - 成功时返回实际解码出的字符数量
  - 失败时返回 `-1`

## 测试覆盖

- 单字符编码顺序检查
- 多字符往返编码/解码
- 非法输入与缓冲区不足
- 起始位、终止位、空闲位错误检测
- 电压阈值判断：负电压为 1，非负电压为 0
