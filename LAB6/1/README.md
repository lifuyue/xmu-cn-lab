# LAB6/1

使用 C++ 模拟二端口网桥的自学习与转发功能。

## 输入格式

标准输入第一行为帧数量 `n`。

接下来 `n` 行，每行输入：

```text
<src_addr> <src_port> <dst_addr>
```

- `src_addr`、`dst_addr` 为 4 bit 地址，支持 `A`、`F` 或 `0xA`、`0xF` 形式
- `src_port` 只能为 `1` 或 `2`
- `0xF` 表示广播地址

## 转发规则

- 收到帧后先学习 `源地址 -> 源端口`
- 目的地址为 `0xF` 时，向另一端口广播
- 目的地址已知且位于另一端口时，定向转发到该端口
- 目的地址未知时，向另一端口泛洪
- 目的地址已知且与源端口相同，则过滤，不转发

程序中的 `Decision` 字段表示：

- `port 1` / `port 2`：定向转发
- `broadcast -> port x`：广播到另一端口
- `flood -> port x`：未知单播，向另一端口泛洪
- `filter`：不过桥

## 编译运行

```bash
make
./bridge_demo
```

示例：

```bash
printf "5\n1 1 2\n2 2 1\n3 1 F\n2 1 3\n2 2 2\n" | ./bridge_demo
```

## 输出内容

- 每帧的源地址、目的地址和转发决策
- 当前自学习得到的 MAC 地址表

MAC 地址表按地址升序输出，广播地址不会被学习进表。
